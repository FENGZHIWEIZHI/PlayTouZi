# 🎲 吹牛骰子 - Liar's Dice

基于 JavaFX 开发的吹牛骰子（Liar's Dice）桌面游戏，支持 1 个真人玩家对战 1-5 个 AI 机器人，内置智能 AI、**AI 大模型对战**、教练模式概率提示、成就系统和持久化玩家档案。

> 启动游戏后即可体验完整功能

## ✨ 功能特色

### 🎮 核心玩法
- **经典吹牛骰子规则**：每人 5 颗骰子，轮流叫牌或开牌
- **万能 1 点**：1 点可以代表任何数字（混子）
- **多种游戏模式**：目标分赛、局数赛、淘汰赛
- **2-6 人对战**：1 个真人 + 多个 AI 机器人

### 🤖 智能 AI（三种模式）

| 模式 | 说明 |
|------|------|
| **规则 AI** | 基于概率计算和固定策略的本地机器人，无需网络 |
| **AI 大模型** | 接入 mimo-v2.5-pro 大语言模型，机器人具备更强的推理和博弈能力 |
| **混合模式** | 优先使用大模型决策，失败时自动回退到规则 AI，推荐使用 |

- 规则 AI：基于正态分布概率评估，根据骰子和叫牌动态决策
- AI 大模型：通过 Prompt Engineering 将游戏局面发送给 LLM，获得智能决策
- 所有模式下机器人决策都具有随机性，增加不可预测性

### 🧠 AI 大模型配置

在游戏设置界面底部可配置 AI 模式：

1. **选择 AI 模式**：规则 AI / AI 大模型 / 混合模式
2. **输入 API Key**：填入 mimo 平台的 API Key
3. **测试连接**：点击「🔗 测试连接」验证 API 是否可用
4. **开始游戏**：机器人将使用大模型进行智能决策

**支持的模型**：mimo-v2.5-pro（兼容 OpenAI 接口协议）

**配置文件位置**：`~/.playtouzi/config.properties`（首次保存后自动生成）

### 🎓 教练模式（可开关）
- **实时概率提示**：叫牌时显示当前叫法的理论概率（高/中/低/极低）
- **智能建议**：根据概率给出「建议开牌」或「建议继续叫牌」的策略
- **回合回顾**：每局开牌后分析"如果当时开牌，胜率是多少"
- 高手可在设置中关闭，或在对局中随时切换

### 📊 全局统计（跨局持久化）
- 总对局数、胜率、最高分、最大连胜
- 叫牌成功率、开牌成功率
- 精准叫牌次数、诈唬成功率
- 1 点战术使用频率、识破诈唬次数
- 数据保存在本地文件，关闭游戏后不丢失

### 🎨 皮肤系统
- **台面皮肤**：经典绿、海洋蓝、皇家紫、烈焰红、黄金奢华
- **骰盅皮肤**：木杯、金杯、水晶杯、龙纹杯、翡翠杯
- 通过解锁成就自动获得对应皮肤

### 💯 完整计分规则
- **基础积分**：赢家 +1，输家 -1
- **精准叫牌**：叫牌数量与实际完全一致 +3 分
- **极限施压**：叫牌数量比实际多 1 个，虽输但 +1 分
- **成功诈唬**：叫牌数量比实际多 3 个以上，开牌者 -2 分，叫牌者 +2 分
- **1 点转换战术**：叫 1 点并获胜 +1 分
- **连胜奖励**：3 连胜 +2、5 连胜 +5、7 连胜 +10、10 连胜直接获胜
- **违规惩罚**：违规叫牌 -2 分、超时 -1 分、误开 -2 分

## 🚀 快速开始

### 环境要求
- **Java 21** 或更高版本
- **Gradle 8.5**（项目已包含 Gradle Wrapper，无需单独安装）
- **网络连接**（仅 AI 大模型/混合模式需要）

### 运行方式

```bash
# 克隆项目
git clone https://github.com/FENGZHIWEIZHI/PlayTouZi.git
cd PlayTouZi

# 运行游戏
gradlew.bat run
```

### 构建项目

```bash
# 编译
gradlew.bat build

# 生成 IntelliJ IDEA 项目文件
gradlew.bat idea
```

## 🎯 游戏规则

### 基本规则
1. 每人摇 5 颗骰子，只能看自己的骰子
2. 按顺序轮流叫牌，例如「6 个 4」表示所有人的骰子中至少有 6 个 4 点
3. **1 点是万能牌**，可以代表任何点数
4. 叫牌必须比上家更高（数量更多，或同数量点数更大）
5. 不相信上家可以喊「开」，所有人亮出骰子验证
6. 如果实际数量达到叫牌数量，开牌者输；反之叫牌者输

### 叫牌规则
- 从普通点数叫到 1 点：数量 × 2 必须 ≥ 上家数量
- 从 1 点叫到普通点数：数量必须 > 上家数量 × 2
- 同类型叫牌：数量相同则点数必须更大
